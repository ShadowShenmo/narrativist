# 导出阅读笔记

> 用途：阅读过程中增量导出 + 全书完成后完整导出 | 被引用：mode-standard-chapter, mode-library, Final Summary

## 增量导出（每章完成后自动追加）

**每读完一章，立即将该章记录追加到 running journal 文件**。不等到全书完成才写入——防止会话中断导致笔记丢失。

### 文件路径

```
output/{sha}/reading-journal.md        # 单书模式
output/{sha}/book{N}/reading-journal.md # library 模式（每书独立）
```

### 追加格式

每章完成后，在文件末尾追加：

```markdown
---

## 第{N}章：{章节名}

**引导角度**：{模板中文名}
**关键词**：{关键词}

**引导问题**：{问题}

**你的回答**：
> {用户回答，或"（跳过）"}

**AI 参考**：
> 证据：{引文}
> 分析：{分析}
> 延伸：{延伸}

**读后一问**：{问题}
**测验**：{题目概要 + 答案}
```

### 首次写入

第一次追加时，先写入文件头：

```markdown
# {书名} · 阅读笔记
> 作者：{作者}
> 开始阅读：{日期}
> 进度：进行中

---
```

## 完整导出（全书完成后）

全书读完后，在增量导出的基础上，补充以下内容到同一文件：

```markdown
---

# 全书总结

## 人物长廊
{各角色演变卡片}

## 章节思考锚点
{汇总表}

## 主题脉络
{全书追问 + 立场地图}

## 你的最终感想
（留白）

---
> 阅读完成：{日期}
> 此笔记由 narrativist v2.3 自动生成
```

## Library 模式导出

- 每读完一本书 → 该书的 `book{N}/reading-journal.md` 已通过增量导出完成
- 合集总结时 → 生成 `journal-index.md`，包含所有书的链接和进度总览

## 文件命名

确保导出文件名不冲突（如已存在则追加数字后缀，如 `reading-journal-2.md`）。导出后告知用户路径。

## 手动导出

用户可随时说"导出笔记"触发导出，不等到全书完成。
